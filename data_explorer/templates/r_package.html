R Package

This page signposts to the climatehealth R package (available via the SOSCHI GitHub or the CRAN repository in Rstudio). It also includes guidance to help users understand the two options available to run the analysis of the SOSCHI indicators, depending on your level of technical experience and the flexibility required.

Both approaches start with the same input - a prepared excel (.csv) file following the data template and data preparation guidance as per the indicator methodology documentation (see sections 2.3 and 3.2, respectively).

Please choose one of the two paths:

Important information:

Intended users: Users who can read, edit and run scripts in R to use pre-built functions.

Requires R installation and installing climatehealth package - please refer to {{ link.external_link('https://github.com/onssoschi/climatehealth', 'climatehealth CRAN documentation.') }}

Key features:

  • Run the full analysis using developed analysis functions for each indicator
  • Allows users to configure analysis parameters
  • Includes descriptive statistics function
  • Statistical model validation is included for selected indicators
Important information:

Important notes

Guidance on sensitivity analysis and using model validation information to iterate development is provided in section 3.6 and 3.7 of the methodology document.

See section 6.2 of the methodology document for guidance on function parameters.

Important information:

Intended users: Users who do not want to write analysis code and wish to run a limited set of indicator analysis using a web interface.

Key features:

  • Runs a predefined, baseline statistical model
  • Currently available for suicides attributable to extreme temperatures
  • Currently available for mortality attributable to wildfire-related PM2.5
  • Currently available for malaria incident cases attributable to extreme temperature and extreme precipitation
Important information:

Important notes

Please note that these indicator tools are still in development, so there may be some inconsistencies between the methods included in the publicly available GitHub code/R package, and those described in the corresponding SOSCHI topic document(s).

It is strongly advised to run descriptive statistics separately following guidelines set in the methods documentation (section 3.3).

Statistical model validation is not available in the online tool (see section 3.7 of methodology document).

See section 6.1 of the methodology document for more information on using the indicator tool.
